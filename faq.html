Toutes les reponses sur MatchPro IA, le machine learning Jokil, les cles API et l'utilisation du logiciel.

Machine Learning & Moteur Jokil

Qu'est-ce que le moteur Jokil ?

Jokil est le moteur de machine learning XGBoost integre a MatchPro IA. Il est pre-entraine sur 10000+ matchings reels avant meme votre installation. Son role: predire avec 85%+ de confiance si un matching candidat-offre va aboutir, en analysant 16 features (competences, experience, distance, teletravail, etc.). Jokil s'ameliore ensuite en continu avec votre usage pour s'adapter a VOS criteres specifiques.

Le logiciel arrive-t-il vide ou deja entraine ?

Le logiciel arrive avec une base de donnees vide (pas de candidats/offres pre-charges), MAIS avec le moteur ML Jokil deja entraine sur 10000+ matchings. Des votre premier matching, Jokil est deja performant. Vous n'avez pas a attendre des mois pour avoir des predictions fiables, contrairement aux autres solutions IA qui partent de zero.

Comment Jokil s'ameliore-t-il avec mon usage ?

Chaque fois que vous acceptez ou refusez un matching, Jokil enregistre cette decision comme un "feedback". Ces feedbacks enrichissent les donnees d'entrainement. Regulierement, Jokil se re-entraine automatiquement pour affiner ses predictions selon VOS criteres (votre secteur, vos exigences salariales, vos preferences geographiques, etc.). Plus vous utilisez MatchPro IA, plus il devient precis pour VOUS.

Que sont les 16 features analysees par Jokil ?

Jokil analyse 16 dimensions par matching: (1) Score semantique, (2) Score regles metier, (3) Ratio experience candidat/requise, (4) Ratio competences matchees, (5) Competences obligatoires matchees, (6) Niveau moyen des competences, (7) Nombre d'experiences, (8) Stabilite du parcours, (9) Distance geographique, (10) Permis, (11) Vehicule, (12) Teletravail, (13) Urgence recrutement, (14) Disponibilite, (15) Type de contrat, (16) Compatibilite metier via dictionnaire Jokil.

Cles API & Providers IA

Comment recuperer une cle API ?

OpenAI (GPT): Allez sur platform.openai.com/api-keys → Creer une cle → Copier

Anthropic Claude: console.anthropic.com → API Keys → Create Key

Google Gemini: aistudio.google.com/apikey → Get API key

Mistral AI: console.mistral.ai → API keys → Create new key

Ollama: Pas de cle requise. Installer depuis ollama.com, endpoint local: http://localhost:11434

Comment configurer Ollama pour fonctionner 100% hors-ligne ?

Etape 1: Telecharger et installer Ollama depuis ollama.com
Etape 2: Ouvrir un terminal et telecharger un modele: ollama pull llama3.2
Etape 3: Dans MatchPro IA → Parametres → API → Ajouter un provider → Ollama
Etape 4: Endpoint: http://localhost:11434, Modele: llama3.2
Etape 5: Activer et mettre en priorite haute (80-100)
Resultat: Aucune donnee ne quitte votre PC. Parsing CV, analyse et matching 100% locaux.

Dois-je payer les cles API en plus de l'abonnement MatchPro IA ?

Oui. MatchPro IA facture UNIQUEMENT l'abonnement logiciel (9.99€ ou 49.99€/mois). Vous fournissez vos propres cles API et payez directement aux providers (OpenAI, Claude, etc.) selon votre consommation. Alternative gratuite: Utilisez Ollama (IA locale, zero cout API, zero donnee partagee).

A quel moment la cle API est-elle sollicitee ?

Cle API utilisee: Parsing CV (extraction texte → donnees structurees), Analyse candidat (seniorite, metier, employabilite), Analyse offre (extraction salaire, teletravail, avantages). Cle API NON utilisee: Matching candidat-offre (Sentence-BERT local), Calcul scores semantiques (embeddings caches localement), Regles metier (100% local), Predictions ML Jokil (XGBoost local). Avec Ollama: Tout est local, zero appel API externe.

Confidentialite & Donnees

Mes donnees sont-elles partagees avec les fournisseurs IA ?

Lors du parsing CV et analyse: Oui, les donnees (texte du CV, description offre) sont envoyees au provider IA choisi (OpenAI, Claude, etc.) pour extraction. Ces providers ont leurs propres politiques de confidentialite. Lors du matching: Non, tout est local (Sentence-BERT + regles + ML). Aucune donnee envoyee. Avec Ollama: Rien ne quitte votre PC, jamais.

Ou sont stockees mes donnees ?

Toutes vos donnees sont stockees localement sur votre PC dans une base SQLite (candidats, offres, matchings, entreprises). Le modele ML Jokil (ml_model.pkl) et les donnees d'entrainement (training_data.json) sont egalement locaux. Vos cles API sont chiffrees AES-256 et stockees localement. MatchPro IA n'a aucun serveur cloud, aucune base distante. Vos donnees vous appartiennent a 100%.

Puis-je utiliser MatchPro IA sans connexion internet ?

Oui, partiellement. Avec Ollama configure: Parsing CV, analyse et matching fonctionnent 100% hors-ligne. Sans Ollama: Le matching fonctionne hors-ligne (Sentence-BERT + regles + ML locaux), mais le parsing CV et l'analyse necessitent internet pour appeler les API cloud.

Utilisation & Fonctionnalites

Comment fonctionne le parsing CV ?

Importez un CV (PDF, DOCX ou image). MatchPro IA extrait automatiquement: nom, email, telephone, adresse, experiences professionnelles, competences techniques et soft skills, formations, diplomes, langues, permis, vehicule. Pour les images, un OCR Tesseract extrait d'abord le texte. L'IA structuree (provider choisi ou Ollama) parse ensuite le texte en donnees exploitables. Le profil est automatiquement enrichi par l'analyse Joskil (seniorite, metier, employabilite).

Qu'est-ce que l'analyse Joskil d'un candidat ?

Lors de l'import CV, Joskil analyse automatiquement: Niveau de seniorite (Junior/Confirme/Senior/Expert), Secteur dominant (IT, commerce, sante, etc.), Metier principal via dictionnaire Jokil, Score d'employabilite (base sur competences, stabilite, polyvalence), Stabilite du parcours (stable/mobile/transition), Recommandations de types d'offres a cibler. Ces infos enrichissent le profil et optimisent les matchings.

Qu'est-ce que l'analyse Joskil d'une offre ?

Lors de la creation d'une offre, Joskil extrait automatiquement depuis la description: Remuneration (min/max, brut/net, periode), Teletravail (jours/semaine, %, full remote), Avantages (tickets resto, mutuelle, 13e mois, primes, etc.), Competences requises (techniques + soft skills, niveaux, obligatoires), Experience (annees min/max, diplomes), Details poste (urgence, taille equipe, management, horaires, deplacements), Langues requises. Ces infos structurees ameliorent la precision des matchings.

Comment interpreter les scores de matching ?

Chaque matching affiche 4 scores: Score global (0-100%) = moyenne ponderee (40% semantique + 40% regles + 20% ML), Score semantique = similarite NLP entre profil et offre, Score regles = competences matchees, experience, distance, teletravail, Score ML Jokil = prediction XGBoost de reussite. Un score global >85% = excellent match, >70% = bon match, >50% = match correct, <50% = faible. Vous pouvez filtrer par score minimum.

Difference entre matching candidat→offre et offre→candidat ?

Candidat → Offre: Vous selectionnez un candidat et MatchPro IA trouve les meilleures offres pour ce profil. Ideal pour recherche d'emploi ou placement. Offre → Candidat: Vous selectionnez une offre et MatchPro IA trouve les meilleurs candidats pour ce poste. Ideal pour recrutement. Dans les deux cas, l'algorithme est identique (semantique + regles + ML). Seul le point de depart change.

Puis-je exporter les resultats de matching ?

Oui. Tous les matchings peuvent etre exportes en CSV avec details complets: scores (global, semantique, regles, ML), competences matchees, distance, disponibilite, etc. Bouton "Export CSV" disponible sur la page resultats. Ideal pour analyses RH externes, rapports, ou import dans votre ATS.
